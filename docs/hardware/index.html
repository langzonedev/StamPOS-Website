Hardware

Build a predictable and supportable StamPOS workstation.

Use known hardware combinations and validate the full sale-to-receipt workflow before an event.

System requirements

Use a 64-bit Windows 11 Pro device with at least 8 GB RAM, SSD storage, 10 GB free disk space and a touch-friendly display with a minimum resolution of 1366 x 768.

Windows terminal

Use a reliable Windows device with current security updates and a stable network connection where card payments are required. Disable unplanned sleep during trading while retaining appropriate screen-lock and security controls.

Receipt printer

Install the manufacturer-supported Windows driver where the selected printer workflow requires it. Confirm paper size, print density, character encoding, cash-drawer pulse settings and automatic cutting. Print a complete sample receipt rather than relying only on a Windows test page.

Cash drawer

Where the drawer connects through the receipt printer, verify the correct port and pulse configuration. Confirm that opening behaviour matches the intended workflow and that users cannot trigger the drawer outside authorised operations.

Payment terminal

For Linkly-connected EFTPOS, follow the approved pairing and merchant setup process. Record the terminal identity and device location, and keep production credentials secure.

Network

Core StamPOS data is local, but connected card transactions require reliable access to the configured payment service. Test the actual event network, including firewall, DNS and captive-portal behaviour, and plan a fallback for network outages.

Pre-event readiness check

Confirm power, charging, network, printer paper, spare paper, cash drawer, application version, backups, operator accounts, products, receipt details, payment logon and a controlled test transaction.

Supported hardware

Use hardware combinations listed in the support matrix, or contact StamPOS to confirm compatibility before deployment.
