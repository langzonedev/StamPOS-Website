Hardware

Build a predictable and supportable StamPOS workstation.

Use known hardware combinations, document every connection and validate the full sale-to-receipt workflow before an event.

Windows terminal

Use a supported Windows 11 device with reliable storage, current security updates, a stable network connection where card payments are required and enough screen space for the touch interface. Disable unplanned sleep during trading while retaining appropriate screen-lock and security controls.

Receipt printer

Install the manufacturer-supported Windows driver where the selected printer workflow requires it. Confirm paper size, print density, character encoding, cash-drawer pulse settings and automatic cutting. Print a complete sample receipt rather than relying only on a Windows test page.

Cash drawer

Where the drawer connects through the receipt printer, verify the correct port and pulse configuration. Confirm that opening behaviour matches the intended workflow and that users cannot trigger the drawer outside authorised operations.

Payment terminal

For Linkly-connected EFTPOS, follow the approved pairing and merchant setup process. Record the logical terminal identity, device location and responsible merchant account. Do not reuse production credentials in demonstrations or unrelated test environments.

Network

Core StamPOS data is local, but connected card transactions require reliable access to the configured payment service. Test the actual event network, including firewall, DNS and captive-portal behaviour. Document the fallback procedure for a network outage.

Pre-event readiness check

Confirm power, charging, network, printer paper, spare paper, cash drawer, application version, backups, operator accounts, products, receipt details, payment logon and a controlled test transaction. Retain a simple equipment map showing which terminal, printer and payment device belong together.

Supported-hardware status

A device appearing technically compatible does not automatically mean it has been fully validated. Production deployments should use hardware combinations confirmed by StamPOS or tested under an agreed pilot plan.
