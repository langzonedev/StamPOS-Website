Core features

Designed around the operator at the counter.

StamPOS focuses on practical selling workflows instead of cluttered screens and unnecessary enterprise complexity.

Fast checkout

Simple sales flow for busy service periods, pop-up stalls and small retail counters.

Offline cash sales

Local desktop operation helps keep the sales process moving when internet access is unreliable.

Receipt printing

Built with receipt output in mind for real POS environments and customer records.

Admin controls

Technician and admin areas support setup, configuration and payment integration workflows.

EFTPOS integration

Payment-aware design for Linkly workflows.

StamPOS is being shaped around Linkly Cloud REST API support, including pairing, payment status handling, recovery and receipt preservation.

Approved

Successful payment states should be clear and immediately actionable.

Unknown

Network failures must be treated as unknown until recovered or reconciled.

Declined

Declined payments should be visually distinct from communication failures.

Cash handling

Built around Australian note denominations.

Operators can press denominations repeatedly, build a running tendered total and see the change due without mental arithmetic during a busy sale.

Events and end-of-day

Support for trading days that do not always end neatly.

StamPOS can evolve cash-up and end-of-day workflows so multi-day events can be counted and reconciled without forcing a final event closure too early.
